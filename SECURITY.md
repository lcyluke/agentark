# Security Policy

## Reporting a Vulnerability

We take the security of Apex seriously. If you discover a security vulnerability, please report it responsibly.

**Do NOT report security vulnerabilities via public GitHub issues.** Instead, please email the maintainers directly at **security@apex.dev** (or use GitHub's private vulnerability reporting feature if enabled).

Please include the following details in your report:

- Description of the vulnerability
- Steps to reproduce
- Affected versions
- Potential impact
- Any suggested mitigation (if known)

## Supported Versions

We currently provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Disclosure Timeline

We aim to follow a coordinated disclosure process:

1. **Acknowledgment** — We will acknowledge receipt of your report within 48 hours.
2. **Assessment** — We will assess the vulnerability and develop a fix within 7 business days.
3. **Release** — A patched version will be released as soon as possible, typically within 14 days of acknowledgment.
4. **Public Disclosure** — We will publish a security advisory after the fix has been released.

We appreciate your help in keeping Apex and its users safe.
