## Development Methodology (Superpowers-Adapted)

This agent follows the Superpowers-inspired development methodology:

1. **🧠 brainstorm first** — NEVER write code without first exploring requirements and getting design approval. Understand the full-stack implications (frontend, API, database).
2. **📝 write a plan** — Decompose into bite-sized tasks with exact paths, code, and assertions. Include frontend components, API endpoints, database migrations.
3. **🔄 test-driven development** — RED-GREEN-REFACTOR: failing test first, then minimal code, then refactor. Apply across all layers (unit, integration, e2e).
4. **🔬 verify before completion** — Fresh verification evidence required before ANY completion claim. Run tests, check API responses, verify UI renders correctly.
5. **🔍 systematic debugging** — NO FIXES without root cause investigation first (4-phase protocol: reproduce → isolate → identify root cause → fix).
6. **👀 request code review** — Always get a reviewer subagent before merging. Both frontend and backend changes need review.
7. **✅ finish properly** — Structured merge/PR/discard decision, never just 'done'. Clean up branches, update docs, verify deployment.

## Code Review Protocol (2-Stage)

This agent follows a 2-stage review process:

1. **Spec Compliance Review** — Does the code match what was spec'd? No more, no less.
2. **Code Quality Review** — Is the code well-structured, tested, and maintainable?

Rules:
- Stage 2 never starts before Stage 1 passes
- Critical findings block merge
- Important findings must be fixed before proceeding
- Minor findings can be noted for later

## Receiving Review Protocol

When receiving code review feedback:

1. **Understand** — Read the feedback fully before responding
2. **Verify** — Is the feedback technically correct? Test it.
3. **Respond** — Accept (implement), push back (with evidence), or negotiate (with alternative)
4. **NEVER** blindly implement reviewer suggestions without understanding why
5. **NEVER** dismiss valid feedback without evidence
