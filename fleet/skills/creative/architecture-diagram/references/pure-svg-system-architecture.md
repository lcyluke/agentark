# Pure SVG System Architecture Diagrams

When the HTML-wrapped architecture diagram template doesn't fit — for standalone SVG files embedded in READMEs, docs, or printed artifacts — use pure SVG with these patterns.

## Use Case

- **README architecture diagrams**: `<picture>` element embedding standalone `.svg`
- **Documentation portals**: Sphinx, Docusaurus, or Mintlify that accept SVG directly
- **Printed/Slides**: no browser dependencies, exact pixel control
- **Very large diagrams**: 1200x1800px+ with 50+ labeled components

## SVG Scaffold

### Document Shell

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 1200 1800" font-family="'SF Mono', 'Courier New', monospace">
  <defs>
    <!-- gradients, filters, markers -->
  </defs>
  <rect width="1200" height="1800" fill="#0a0e27"/>
  <!-- content -->
</svg>
```

### Layer-Group Structure (5-Layer System Architecture)

```
L5 — Interface Layer
L4 — Intelligence Layer  
L3 — Orchestration Layer
L2 — Agent Runtime Layer
L1 — Provider Layer
[Cross-cutting: Token Economy]
[Data Flow Legend]
```

Each layer is a rounded `<rect>` with:
- Gradient fill (e.g. `#1e3a5f` → `#162d4a`)
- 1.5px colored stroke (unique per layer)
- filter="url(#shadow)" for depth

### Color Palette

| Layer | Gradient | Stroke |
|-------|----------|--------|
| L5 Interface | `#1e3a5f→#162d4a` | `#3b82f6` (blue-500) |
| L4 Intelligence | `#2a1a4a→#1f1236` | `#8b5cf6` (violet-500) |
| L3 Orchestration | `#1a3a3a→#132d2d` | `#10b981` (emerald-500) |
| L2 Agent Runtime | `#1e3a5f→#162d4a` | `#f59e0b` (amber-500) |
| L1 Provider | `#3a1a2a→#2a1220` | `#ef4444` (red-500) |
| Cross-cutting | amber→orange gradient | — |

### Component Box Pattern

Each component inside a layer:

```svg
<rect x="65" y="168" width="240" height="95" rx="8" fill="#1e293b" stroke="#334155" stroke-width="1"/>
<text x="185" y="190" text-anchor="middle" fill="#e2e8f0" font-size="14" font-weight="bold">Component Title</text>
<text x="185" y="208" text-anchor="middle" fill="#94a3b8" font-size="10">Subtitle line</text>
<text x="185" y="224" text-anchor="middle" fill="#64748b" font-size="9">Detail line 1</text>
<text x="185" y="240" text-anchor="middle" fill="#64748b" font-size="9">Detail line 2</text>
```

### Mini-badges Inside Components

```svg
<rect x="100" y="492" width="120" height="24" rx="4" fill="#2d1b69" stroke="#7c3aed" stroke-width="0.5"/>
<text x="160" y="509" text-anchor="middle" fill="#a78bfa" font-size="9">Badge Text</text>
```

### Layer-to-Layer Arrows

```svg
<line x1="600" y1="280" x2="600" y2="310" stroke="#4b5563" stroke-width="2" marker-end="url(#arrowDown)"/>
```

Define arrow markers in `<defs>`:

```svg
<marker id="arrowDown" markerWidth="10" markerHeight="8" refX="5" refY="4" orient="auto">
  <path d="M0,0 L10,4 L0,8 Z" fill="#6b7280"/>
</marker>
```

### Data Flow Legend at Bottom

Best practice: a framed bottom section showing the request-response round-trip:

```
User Input → L5 Interface → L3 Mode → L3 Orchestrate → L2 Runtime → L1 Provider
                                                                                ↓
User Output ← L5 Interface ← L3 Result ← L2 Return ← ← ← ← ← ← ← ← ← ← ← ← ← ←
                      ↑
                L4 Feedback (Evolution + Knowledge Graph learn from every execution)
```

### Shadows (Gradient Drop)

```svg
<filter id="shadow">
  <feDropShadow dx="0" dy="2" stdDeviation="4" flood-color="#000" flood-opacity="0.4"/>
</filter>
<filter id="glow">
  <feGaussianBlur stdDeviation="3" result="blur"/>
  <feMerge><feMergeNode in="blur"/><feMergeNode in="SourceGraphic"/></feMerge>
</filter>
```

### Title & Layout

```svg
<text x="600" y="55" text-anchor="middle" fill="#e2e8f0" font-size="28" font-weight="bold" letter-spacing="2">System Title</text>
<text x="600" y="80" text-anchor="middle" fill="#94a3b8" font-size="14">Subtitle / Tagline</text>
```

## Embedding in README

Use the `<picture>` element for dark/light mode adaptivity:

```html
<picture>
  <source media="(prefers-color-scheme: dark)" srcset="https://raw.githubusercontent.com/user/repo/main/docs/images/architecture.svg">
  <img alt="Architecture" src="https://raw.githubusercontent.com/user/repo/main/docs/images/architecture.svg" width="100%">
</picture>
```

## When to Use HTML-Wrapped vs Pure SVG

| Decision | HTML-Wrapped | Pure SVG |
|----------|-------------|----------|
| Component icons (emoji/symbols) | ✅ Include in HTML divs | ✅ Embed in `<text>` nodes |
| Interactive hover/click | ✅ CSS hover works | ❌ No interactivity |
| README embedding | ✅ Via iframe | ✅ Direct `<img>` or `<picture>` embedding |
| Print/offline | ✅ Single file | ✅ Single file |
| Very large (50+ boxes) | ⚠️ Slow rendering | ✅ Fast, lightweight |
| CI/CD auto-generation | ⚠️ Needs browser | ✅ Can be generated by any tool |
