# 微信小程序技术架构决策指南

> 适用场景: 用户问「出技术方案」或「小程序选什么框架」时加载。
> 核心原则: 5h/周副业 → 不是选最好看的架构，是选最快上线的。

---

## 框架选型决策树

```
用户想做微信小程序
│
├─ 只做微信？ ─── YES ──→ ✅ 微信原生 (WXML+WXSS+JS)
│                         包最小 / 调试最快 / 零编译层
│
├─ 以后可能扩抖音/支付宝？ ──→ ⚠️ uni-app (Vue) 或 Taro (React)
│                           投资一套代码跑多端
│                           代价: 多一层编译调试 / 基础包体积大
│
└─ 已有Web项目想转小程序？ ──→ kbone (腾讯出品，Web→小程序)
```

### 不推荐的方案

| 方案 | 不选理由 |
|:----|:---------|
| uni-app + 只做微信 | 跨端能力过剩，学框架的时间 ≈ 上线时间 |
| Taro + 不熟悉React | 为了用Taro学React ≈ 本末倒置 |
| 微信云开发 | 已有自购服务器就不要再加一层 |
| Docker/K8s | 一人项目，加容器=加维护 |
| 前端AI推理 | 小程序包限制2MB，模型放服务端 |

## 3页轻量方案模板

微信小程序 MVP 的最简文档结构：

```
1. 技术选型（1句话 + 理由表）
2. 页面路由（首页→上传→结果，3个页面）
3. API契约（前端调后端的接口定义，参数+响应示例）
4. 部署方案（服务器→nginx→SSL→小程序白名单）
5. 成本清单（服务器+域名+认证）
```

**原则：** 写代码前花90分钟出方案，但不要写UML/ER图/数据字典——3个页面不值得。

## 上线阻塞顺序（关键！）

```
① 注册域名        → 10分钟
② ICP域名备案      → 5-20工作日 (腾讯云代备案)
③ SSL证书          → 30分钟 (Let's Encrypt 免费)
④ nginx HTTPS配置  → 30分钟
⑤ 后端部署+依赖安装 → 2小时
⑥ 小程序域名白名单  → 10分钟
⑦ 真机调试+提交审核 → 1-7天
```

**⚠️ ICP备案是最大阻塞项** — 不能在它之前配HTTPS，不能配HTTPS就不能配白名单，不能配白名单小程序就连不上后端。建议备案和写代码并行。

## 个人小程序 vs 企业小程序

| 维度 | 个人 | 企业 |
|:----|:---:|:----:|
| 认证费 | 0元 | 300元/年 |
| 类目限制 | 有限 | 几乎不限 |
| 微信支付 | ❌ | ✅（需对公账户） |
| 发布类目 | 体育/工具等 | 几乎全部 |
| 适合阶段 | MVP验证 ✅ | 正式运营 |

## 代备案注意事项

腾讯云代备案流程（ICP）：
- 个人备案：身份证+域名证书+居住证（部分省市）
- 备案期间域名不能解析，不能访问
- 审核通过后会有短信通知
- 拿到备案号后配CNAME到服务器IP
- **可以先在DevTools用HTTP开发，不耽误写代码**

## 坑

- 小程序强制HTTPS，不能用裸IP
- nginx 要配 `client_max_body_size 100M` 不然大图 upload 413
- `wx.uploadFile` 的 `name` 参数必须和 FastAPI 的 `File(...)` 参数名一致
- 基础库选 2.33.0，`enhance:false`，避 babel/runtime 打包bug
