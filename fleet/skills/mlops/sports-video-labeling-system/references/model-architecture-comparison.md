# 羽毛球动作评估 · 全模型架构对比

> 分析日期: 2026-06-01 | 基于 172→276 条实际标注数据验证

## 核心结论

**没有一个模型能独自完成全部任务。** 羽毛球视频评估需要多阶段专业模型组合，不是端到端大模型。

## 推荐架构: 多阶段管道

```
视频 -> RTMPose(骨骼) -> VideoMAE(分类) -> GAT+TCN(评估)
     -> DTW(纠错) -> DeepSeek/Qwen(训练建议)
```

| 环节 | 推荐模型 | 大小 | 部署 | 数据需求 |
|:-----|:---------|:---:|:-----|:---:|
| 3D 骨骼 | RTMPose | 15MB | 小程序端 | 0 (预训练) |
| 动作分类 | VideoMAE v2 | 86MB | 腾讯云 | 200微调 |
| 技能评估 | GAT+TCN(或GBDT@<500) | 2MB | 腾讯云 | 500 |
| 关节纠错 | DTW+生物力学 | 代码 | 腾讯云 | L7基准 |
| 伤病风险 | GBDT(已有) | 300KB | 腾讯云 | 已有 |
| 训练建议 | DeepSeek-V3(当前) | — | API | 0 |
| 本地替代 | Qwen2.5-1.5B QLoRA | ~800MB | 腾讯云 | 500对 |

## 信息漏斗问题

```
视频(300MB) -> MediaPipe骨骼(7KB) -> 规则(136B) -> GBDT -> 等级
信息保留率: 136/300,000,000 ≈ 0.00005%
```

GBDT卡在76.1%不是算法差——信息已被压缩到只剩渣。GAT+TCN从骨骼序列直接学习，只需更多数据。
