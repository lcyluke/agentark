# 项目状态汇报模板

当被询问"项目进展如何"时，使用此模板：

```
⚓ [📦 🏸 羽球宝AI搭子] [🎯 PM] [进展汇报]

**当前航位：{版本号} — {一句话状态}**

| 维度 | 状态 |
|:--|:--|
| 📦 代码 | {最新 commit 信息} |
| 🧪 UAT | {通过数/总数} |
| 🧠 模型 | {模型状态} |
| 🎨 前端 | {页面/API 数量} |
| 💳 支付 | {支付对接状态} |
| 🐙 GitHub | {仓库 + push 状态} |
| 🚀 部署 | {服务器/部署状态} |
| 📱 审核 | {微信审核状态} |

**📋 路线图下一站：{当前阶段}**
{简要列出下一阶段关键任务}

**🚧 阻塞点：**
{有则逐一列出，无则省略}
```

## 原则
- 先查再报：先 `git log` + `git status`，再汇报
- 表格优先：状态表一目了然
- 阻塞点必须列出：老卢要看到卡在哪里
- 不超过 25 行：微信聊天场景
