# 跨平台上下文混淆 — 实战案例 (2026-05-30)

## 场景

1. **微信端：** 用户发了一条消息要求分析「IoT 设备之间 CAN 接口通讯安全解决方案」
2. **CLI 端：** 用户打开终端，说「按照你推荐变通方案执行」（接的是之前关于跨平台协作的讨论）
3. 接着在 CLI 做了网关 shutdown 分析
4. **回到微信/CLI：** 用户说「刚才在微信端要求 IoT Agent 分析一份 IoT 设备之间 CAN 接口通讯安全解决方案，不是 shutdown 得报告啊」

## 错误根因

在 CLI 会话中，收到了模糊的 follow-up 指令（「按照你推荐变通方案执行」），我顺着 CLI 上下文（网关 shutdown 事件）继续工作，完全忽略了用户在微信端提出的 CAN 安全分析请求。

**核心问题：** 跨平台会话隔离意味着我不会有微信端的上下文注入到 CLI 会话。当用户说「刚才」时，指的是跨平台的时间维度，而不是同一会话内的时间维度。

## 正确做法

1. 当用户在任一平台说「刚才我让你做 X」或「继续之前那个 Y」时，该任务**可能在其他平台上提出过**
2. 如果当前 Memory 中没有该任务的记录，应该反问：「你是在微信/钉钉上提的这个吗？我这边没看到上下文，能再描述一下吗？」
3. **不要假设**当前会话的上下文就是用户在说的那个上下文

## 避免此类混淆的检查清单

- [ ] 用户的 follow-up 消息是否引用了我不在当前会话中见过的内容？
- [ ] 如果用户说「之前在 XX 平台」，是否先回想/搜索了那个平台的消息？
- [ ] 我生成的回答是否与用户**当前问题**匹配，而不是我**刚做完的工作**？
